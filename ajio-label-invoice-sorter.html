Marketplace Tool · AJIO Packing Workflow

AJIO Label Invoice Sorter

Arrange AJIO labels and invoices into one packing-ready PDF, sort by forward bag barcode, and stamp each label with Seller SKU plus bag barcode or FN order number.

How it works

  • Upload one or multiple AJIO Label PDFs, Invoice PDFs and bulk accepted Excel files.
  • The tool reads Customer Order ID, Seller SKU ID, *Confirm Quantity and Forward Consignment Bar Code from AJIO Excel.
  • Labels are matched by direct FN/AWB when possible. If barcode/text is not enough, customer details from the label are matched with invoice customer details.
  • If Excel is missing for an order but invoice is matched, SKU is taken from invoice and the FN order number is printed instead of bag barcode.
  • If Excel exists but bag barcode is blank, SKU comes from Excel and the FN order number is printed on the barcode line.
  • Final PDF is generated as label → invoice sequence and sorted by bag barcode for faster dispatch table arrangement.

Privacy: No uploads. Everything stays on your device.

Independent utility for marketplace sellers. Not affiliated with AJIO or Reliance Retail.

How to use

  1. Upload all AJIO label PDFs for the orders you want to print.
  2. Upload invoice PDFs when available. Invoice helps with customer-text matching and label → invoice output.
  3. Upload AJIO Excel files. Multiple Excel files are supported for split batches.
  4. Click Generate Final PDF.
  5. Check the validation report. If any label is marked unsafe, verify it manually before packing.

Important rules

  • Default sorting is bag barcode. Orders without bag barcode are placed after barcode orders and sorted by SKU.
  • Multi-quantity SKU stamping is automatically wrapped and scaled inside the bottom-right label area.
  • For 1+ quantity, the tool uses AJIO Excel Column U: *Confirm Quantity.
  • Labels are not skipped. If a label cannot be safely matched, it is still included with manual-check marking.
